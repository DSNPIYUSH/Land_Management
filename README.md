# 🗺️ Land Information Management System (DBMS Portal)

**System Registry ID:** 24EU08079  
**Academic Presentation Package**

An all-in-one academic RDBMS presentation package and interactive web portal modeling a comprehensive **Land Information Management System**. Instead of a plain collection of raw scripts, this workspace packages database engine logic, relational theory, query benchmarks, and coursework assignments into a review-friendly digital dashboard.

---

## 🖥️ Core Workspace Features

* **Interactive Website Portal:** A beautifully styled web interface utilizing glassmorphism effects, custom fade animations, and dedicated sidebar navigation to browse the entire system lifecycle seamlessly.
* **Unified Documentation Hub:** Embedded project abstracts (`abstract.pdf`) and home assignment submissions accessible directly inside the dashboard view layer.
* **Visual Evidence Base:** High-fidelity system execution snapshots mapping DDL structures, constraint behaviors, and aggregate query returns directly from the database engine terminal.

---

## 🗄️ Database Architecture & Schema Design

The backend relational model structures the legal, temporal, and spatial boundaries of land tracking. The environment enforces strict integrity boundaries using `NULL`, `NOT NULL`, `UNIQUE`, `PRIMARY KEY`, and `FOREIGN KEY` constraints across the following entities:

* **Land:** Core spatial assets containing parcel coordinates, acreage, and valuation vectors.
* **Owner:** Individual and corporate identity registers.
* **Ownership & Ownership History:** Direct semantic mapping tables and historical immutable ledgers detailing exact transactional title chains over time.
* **Registration & Registration In:** Official jurisdictional documentation logs, timestamps, and deeds mapping legal registration boundaries.

## 💻 SQL Implementation & Queries

The project workspace serves as an implementation testbed executing a wide array of relational operations divided into clear database operational categories:

### 1. Data Definition Language (DDL)
Schema creation, architectural definitions, and structural tablespace modification tracking:
* `CREATE`, `ALTER`, `DROP`, `MODIFY`, and `CHANGE` structural mutations.

### 2. Data Manipulation Language (DML)
Runtime state manipulation commands documenting data lifecycles:
* `INSERT`, `UPDATE`, and `DELETE` record sequences.

### 3. Advanced Querying Mechanics
Deep-dive data analytics pipelines across relational horizons:
* **Operators & Filtering:** Pattern evaluation via `LIKE`, value grouping with `GROUP BY`, criteria filtration via `HAVING`, and structural Sorting via `ORDER BY`.
* **Nested & Sub-Queries:** Evaluated using advanced relational operations: `IN`, `NOT IN`, `EXISTS`, `ALL`, and `ANY`.
* **Aggregate Functions:** System valuation computations executing math operations over entire columns: `SUM`, `COUNT`, `AVG`, `MAX`, and `MIN`.
